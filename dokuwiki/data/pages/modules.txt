====== Modules ======

Les modules réfèrent dans cette documentation aux applications intégrées à AppFM, au fichiers qui composent ces applications ou plus spécifiquement aux [[:spec_draft|fichiers de description qui définissent le cadre (frame) de ces applications]].

Ainsi les modules sont des programmes associés à une description formalisée qui permet à AppFM d'orchestrer ces programmes entre eux et de les exécuter le plus indépendamment du système sur lequel est installé la plateforme AppFM.

===== Méthode d'encapsulation =====

Fondamentalement, un programme peut être définit et lancé via un script bash prenant en entrée des paramètres et produisant des résultats via la création de fichiers ou tout simplement en inscrivant le résultat dans la sortie standard.

Mis à part quelques opérateurs supplémentaires permettant de lancer en parallèle des modules ou de créer des points de branchement, AppFM exécute simplement ces scripts shell dans un répertoire d'éxécution propre à chaque "run" via le script ''RACINE_INSTALLATION/pshell/bin/cpm-process-shell.py'' qui (éxécuté à l'intérieur d'un conteneur Docker ou nativement depuis le serveur) redirige les sorties (standard et d'erreur) du script et informe AppFM de l'état du programme.

Pour les applications nécessitant un chargement de ressource assez lourd. Il est possible de constituer des modules selon un schéma client-serveur. Pour cela, AppFM permet de construire des services prenant le rôle de serveur et dont le client serait le module dont l'éxécution pourrait par exemple consister en une requête HTTP.
