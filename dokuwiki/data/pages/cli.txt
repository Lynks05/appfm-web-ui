====== Command Line Interface Manual ======

===== Installation =====

<code>
wget URL; unzip cpm-cli.zip; cd cpm-cli; pip install --user .; sh ./generate-autocomplete.sh ; echo ". cpm-cli/cpm-complete.sh">> ~/.bashrc
</code>

then edit //server.conf// to indicate the host and port of a running cpm-core installation
===== Module Management =====

- Reload the module definitions : (if modifications have been made on .module files)

<code>
cpm reload
</code>

- List all available modules

<code>
cpm module ls
</code>

- Run the module //MODULE_NAME// with optional (but often required) configuration file //CONF_FILE// (hosted on server)
<code>
cpm run MODULE_NAME [CONF_FILE]
</code>

returns a <process id>

===== Run Management =====

<code>
cpm process ls [-a]
</code>

# default active, -a all

cpm process run [-i //user_input//] [–from //mid//] [–to //mid//] //pid// |//process_name// |//mid// |//module_name// //corpus_name// |//cid// |//filepath//

cpm process stop //prunid//

# how to view results?

===== Corpus Management =====

cpm corpus create [//rootdirpath//] //name//

# name must be unique

# copy files ? yes

cpm corpus //name// |//id// add //filepath// |//folderpath//

# check if exist

cpm corpus //name// |//id// rm //relativefilepath// |//relativefolderpath//

cpm corpus //name// |//id// ls

cpm corpus ls

cpm corpus rm //name// |//id//
